About & Tutorials

Cyperf-CE Test Controller

Overview

The Cyperf-CE Test Controller is a modern Flask-based web application designed to provide an intuitive and efficient user experience for configuring, running, and monitoring network performance tests using the cyperf-ce network traffic generator.

Key Features
Test Parameters Configuration: Easy-to-use forms for setting up throughput and CPS tests
Real-time Statistics: Live monitoring of test progress with interactive charts
Log Streaming: Real-time client and server log viewing with filtering capabilities
Responsive Design: Works seamlessly across desktop, tablet, and mobile devices
Technology Stack
Backend: Flask (Python)
Frontend: Tailwind CSS
Charts: Chart.js
AJAX: jQuery
Icons: Heroicons (SVG)
API Client: Python Requests

Quick Start Guide

1

Configure Test Parameters

Navigate to the Test Parameters section and configure your server and client IP addresses, test type (throughput or CPS), and other parameters.

2

Start Your Test

Click the "Run Test" button to start your network performance test. The system will automatically start the server and client components.

3

Monitor Statistics

Switch to the Statistics & Graphs section to view real-time performance metrics, charts, and test progress.

4

View Logs

Check the Test Logs section for detailed client and server logs, with filtering and search capabilities.

Running a Throughput Test

Learn how to configure and run a basic throughput test to measure network bandwidth.

  1. Set Test Type to "Throughput Test"
  2. Configure server and client IP addresses
  3. Set target bandwidth (e.g., 100 Mbps)
  4. Adjust packet size if needed (default: 1500 bytes)
  5. Set test duration (default: 60 seconds)
  6. Click "Run Test" to start

Running a CPS Test

Learn how to configure and run a connections per second (CPS) test.

  1. Set Test Type to "Connections Per Second (CPS) Test"
  2. Configure server and client IP addresses
  3. Set target CPS rate (e.g., 1000 CPS)
  4. Set concurrent connections limit
  5. Set test duration
  6. Click "Run Test" to start

REST API Endpoints

POST /api/start_server

Start the cyperf-ce server with test configuration

POST /api/start_client

Start the cyperf-ce client for the specified test

GET /api/server/stats/{test_id}

Get current server statistics for the specified test

GET /api/client/stats/{test_id}

Get current client statistics for the specified test

POST /api/stop_server

Stop and cleanup all cyperf server processes

Common Issues

Test fails to start

Check the following:

  • Verify server and client IP addresses are reachable
  • Ensure cyperf-ce API server is running on port 8000
  • Check firewall settings for the specified ports
  • Verify network connectivity between server and client

No statistics showing

Possible causes:

  • Test may not have started successfully
  • Statistics collection interval may be too high
  • Network issues preventing data collection
  • Browser JavaScript errors (check console)

Charts not displaying

Try these solutions:

  • Refresh the page and try again
  • Check browser compatibility (modern browsers required)
  • Verify Chart.js library is loading properly
  • Clear browser cache and reload

Frequently Asked Questions

What is the difference between throughput and CPS tests?

Throughput tests measure the maximum data transfer rate (bandwidth) between client and server, while CPS (Connections Per Second) tests measure how many new connections can be established per second.

Can I run tests between different networks?

Yes, as long as the server and client IP addresses are reachable from each other and the necessary ports are open in firewalls.

What ports does cyperf-ce use?

The cyperf-ce API server runs on port 8000 by default. Test traffic uses configurable ports (default 5202). The web interface runs on port 5000.

How long can I run a test?

Tests can run from 1 second to 1 hour (3600 seconds). Longer tests provide more stable and accurate results.

Can I export test results?

Yes, you can export statistics data and charts from the Statistics section, and export logs from the Test Logs section.
